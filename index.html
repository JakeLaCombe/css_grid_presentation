We got Flexible

CSS3 gave us the Flexbox which made laying out elements easier on a page

  • Easy access to equal height columns
  • Very easy to build navigation menus width.
  • Prevents a lot of the hacks that had to be done in previous browsers with floats and position styles

Not all can be flexed

Try flexing frozen laffy taffy

Flexbox gave us solutions, but it didn't address all of our concerns.

  • Flexbox was designed to adjust space for content in it's individual container.
  • Flexboxes only go one dimension.
  • This can make it tricky to align items that are on multiple rows/containers in a layout.

Todays Solution - CSS Grid

Introduction to CSS Grid

  • CSS Grid is a specification in CSS3 that allows you declare a layout with 2 dimensions.
  • Declaring a container as a gridded container allows for the alignment of children in regards to rows and columns
A Complete Guide to Grid by CSS Tricks
https://css-tricks.com/snippets/css/complete-guide-grid/

Benefits of a CSS Grid

  • You no longer need any extra containers to get your grid working.
  • Very easy to flex your template on different breakpoints.
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
  </ul>

  ul {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }

Getting Started with the CSS Grid

  • To get started, add display: block to your container.
  • That's all you need to start a one column layout. If you want more columns, set grid-template-columns on the container, specifying a width for every column you wish to have in your grid.
  • Once you have that setup, your gridding is good to go.

CSS Properties - The Parent

CSS Properties - The Parent

  • display: - You have multiple choices for this category. grid, inline-grid, or subgrid
    • subgrid - This display tells a grid inside a grid to inherit it's template from it's parent.

CSS Properties - The Parent

Structural Properties

  • grid-template-columns: How many columns do you want, and how much space you want for each column. (Can be percentages, set widths, and fractals).
    • grid-template-columns: 1fr 2fr 1fr 100px;
    • grid-template-columns: repeat(6, 1fr);
  • grid-template-row: How many row do you want, and how much space you want for each column.
    • grid-template-row: 100px 1fr;
    • grid-template-columns: repeat(3, 1fr);

Terminology Break - Grid Types!

Previous properties define grid type

Implicit Grid
A grid that is infinitely repeating. Set by default, and configured with grid-template-columns.
Explicit Grid
A grid that is finite. To configure this, set grid-template-rows.

(Note: If you supply more cells than space available, any cells after the area turn into an implicit grid).

CSS Properties - The Parent

Structural Explicit Properties

  • grid-auto-columns: Lengths for each auto generated column. Multiple can be provided. (Can be percentages, set widths, and fractals).
    • grid-auto-columns: 1fr 2fr 1fr 100px;
    • grid-auto-columns: repeat(6, 1fr);
  • grid-auto-rows: Lengths for each auto generated row. Multiple can be provided.
    • grid-auto-rows: 100px 1fr;
    • grid-auto-rows: repeat(3, 1fr);

CSS Properties - The Parent

Gaps

  • grid-row-gap: Gap you want between cells on the vertical axis.
  • grid-column-gap: Gap you want between cells on the horzontal axis.
  • grid-gap: Short-handed notation <grid-row-gap grid-column-gap>. A single value can be provided as well that gets applied to both the row and the column gaps.

CSS Properties - The Parent

Alignment of Children

  • align-items: How items should be aligned on the column axis.
    • If you want to align an individual cell, use align-self property on the specific child.
  • justify-items: How items should be aligned on the row axis.
    • If you want to align an individual cell, use justify-self property on the specific child.
  • Values - start | end | center | stretch

CSS Properties - The Parent

Alignment of the actual grid

  • align-content: How the grid should be aligned on the column axis.
  • justify-content: How the grid should be aligned on the row axis.
  • Values - start | end | center | stretch | space-around | space-between | space-evenly

CSS Properties - The Children

CSS Properties - The Children

Positioning Properties

  • grid-row-start: What row the child should be positioned at. Starts at 1.
  • grid-row-end: What row the child should end at. Starts at 1.
  • grid-column-start: What column the child should be positioned at. Starts at 1.
  • grid-column-end: What column the child should end at. Starts at 1.

CSS Properties - The Children

Short Handed Positioning Properties

  • grid-row: - <grid-row-start> / <grid-row-end>.
  • grid-column: - <grid-column-start> / <grid-column-end>.

CSS Properties - The Children

Positioning End

  • The end properties for position do not have to be specific column positions.
  • Custom identifiers can be provided to these values as well.
    • grid-row-end: span 4; would take a cell and span it 4 rows.
    • grid-column-end: span 4; would take a cell and span it 4 columns.

Grid Areas

CSS Properties - The Children

Positioning End

  • In addition to templating the rows and columns of a grid, you can template the entire grid area as well!
  • You essentially give all of your grid cells an identifier, and then tell the cells to go to match those identifiers.
  • To start, set grid-template-areas on the grid, and then set grid-area on the individual children.
  • Note: You can only have an explicit area with grid-area. Any grid items that do not correspond to a cell identifier get placed on the implicit tracts of a grid.

Browser Support

Browser Support for Grid

  • All modern browsers (Chrome, Safari, Firefox, Edge < 16) support CSS Grid implementation
  • IE 10, IE11, and early edge support an old implementation of CSS Grid

IE & Early Edge

Worse than working at Amazon

  • Sadly, grid isn't as easy as just adding vendor prefixes to the grid styles.
  • Syntax of IE and Edge have quite a different pair of mappings.
  • In addition, grids in IE can only be declared explicitly. No infinite grids with the IE implementation.
  • Gaps aren't fully supported as well. To get gaps, you have to include them in your column template.

Should we use it?

  • Yes!!!!
  • Browsers are starting to catch up to the grid implementation.
  • Terra has a solution that assists with the IE grid differences as well. Checkout terra-dynamic-grid.
  • It'll make your markup look much better than it did before!

Flexbox Layout vs Grid Layout

When to use one over the other

It's not a black and white choice.

  • First, remember the definition between the two
    • Flexbox is for having elements take as much space as needed for it's given container
    • Grid is for getting elements to align in a two dimensional system.
  • There are cases for when you want to make sidebars as large as they need to be for example.

Case for Flexbox

Great for Yoga Websites

  • You don't care about aligning elements from line to line.
  • You just want stuff to take as much space as it needs.
  • Your layout goes in just one dimension.

Case for Grid

Great for Flynn and his Son

  • You need elements to align from row to row.
  • Your items span in two dimensions.
